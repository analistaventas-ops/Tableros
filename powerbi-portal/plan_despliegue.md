# Plan de Despliegue (Supabase + Vercel)

Este documento detalla los pasos para pasar la aplicación de un entorno local a uno de producción usando **Supabase** (Base de Datos) y **Vercel** (Frontend y Backend Serverless).

## 1. Configuración de Supabase

1. Crea un proyecto en [Supabase](https://supabase.com/).
2. Copia tu `SUPABASE_URL` y `SUPABASE_ANON_KEY` (y preferiblemente `SUPABASE_SERVICE_ROLE_KEY` del panel de configuracion).
3. Entra al **SQL Editor** y ejecuta el siguiente script para crear las tablas necesarias:

```sql
-- Positions Table
CREATE TABLE IF NOT EXISTS positions (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    name TEXT UNIQUE NOT NULL,
    dashboard_url TEXT NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Users Table
CREATE TABLE IF NOT EXISTS users (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    password_plain TEXT,
    name TEXT NOT NULL,
    email TEXT,
    role TEXT DEFAULT 'user' NOT NULL,
    position_id BIGINT REFERENCES positions(id),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Access Logs Table
CREATE TABLE IF NOT EXISTS access_logs (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    user_id BIGINT NOT NULL REFERENCES users(id),
    login_time TIMESTAMPTZ DEFAULT NOW(),
    dashboard_url TEXT
);
```

## 2. Preparación del Backend

Modificaremos el backend para usar Supabase en lugar de SQLite.

-   Instalar dependencias: `npm install @supabase/supabase-js` en la carpeta `backend`.
-   Actualizar `database.js` para inicializar el cliente de Supabase.
-   Actualizar `server.js` para usar los métodos de Supabase (`db.from('users').select(...)` etc).

## 3. Preparación del Frontend

-   Asegurarse de que las llamadas a la API usen una URL dinámica basada en variables de entorno (por ejemplo, `VITE_API_BASE_URL`).
-   Configurar el build para producción.

## 4. Despliegue en Vercel

### Frontend
1. Conecta tu repositorio de GitHub (que ya iniciaste) a Vercel.
2. Agrega las Variables de Entorno en el panel de Vercel:
    - `VITE_API_BASE_URL`: (La URL que te dé Vercel para el backend, usualmente la misma app).

### Backend (Express en Vercel)
Vercel permite desplegar apps Express como funciones serverless. Crearemos un archivo `vercel.json` en la raíz para configurar esto.
